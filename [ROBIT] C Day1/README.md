## 과제 1. 평균, 최댓값, 최솟값 구하기

### 구현 방법

사용자로부터 다섯 개의 실수를 입력받아 평균, 최댓값, 최솟값을 출력하는 프로그램을 작성하였다. `for` 반복문으로 입력을 다섯 번 받고, 입력값을 누적한 뒤 5로 나누어 평균을 계산하였다.

첫 번째 입력값을 최댓값과 최솟값의 초기값으로 저장하고 이후 입력값과 비교하여 값을 변경하였다.

### 어려웠던 점 및 해결 방법

이 과제에서는 크게 어려웠던 점은 없었다. `for` 반복문으로 입력을 반복하고, `if` 조건문으로 입력값의 크기를 비교하는 방법을 사용하여 손쉽게 해결할 수 있었다.

## 과제 2. 피보나치 수열 구하기

### 구현 방법

사용자로부터 `n`을 입력받아 n번째 피보나치 수를 출력하였다. 이전 항을 저장하는 `a`, 현재 항을 저장하는 `b`, 두 항의 합을 저장하는 `next` 변수를 사용하였다.

반복문이 실행될 때마다 앞의 두 수를 더하고, 변수의 값을 다음 항으로 변경하였다. 또한 2보다 작은 수가 입력되면 안내 메시지를 출력하였다.

### 어려웠던 점 및 해결 방법

처음에는 피보나치 수열의 모든 값을 배열이나 리스트 형태로 저장하여 문제를 해결하려고 하였다. 하지만 수업에서 아직 배열이나 리스트를 배우지 않았기 때문에 다른 방법을 고민하는 데 시간이 걸렸다.

각 항을 모두 저장하지 않고 이전 항과 현재 항만 있으면 다음 항을 계산할 수 있다는 점을 이용하였다. `a`, `b`, `next` 세 개의 변수에 필요한 값만 저장하고 반복문이 실행될 때마다 값을 갱신하는 방법으로 해결하였다.

## 과제 3. 윤년 판별하기

### 구현 방법

입력받은 연도가 윤년인지 조건문을 이용하여 판별하였다. 연도가 4의 배수이면서 100의 배수가 아니거나, 400의 배수이면 윤년으로 출력하였다.

조건을 만족하지 않는 경우에는 윤년이 아니라는 메시지를 출력하였다.

### 어려웠던 점 및 해결 방법

윤년을 판별하는 공식이 명확하게 정해져 있어 크게 어려웠던 점은 없었다. 나머지 연산자 `%`를 사용하여 배수 여부를 확인하고, `if`, `else if`, `else` 조건문을 사용하여 각 조건을 구분함으로써 손쉽게 해결할 수 있었다.

## 과제 4. 사칙연산 계산기

### 구현 방법

두 개의 실수와 연산자를 입력받아 덧셈, 뺄셈, 곱셈, 나눗셈, 거듭제곱을 계산하는 프로그램을 작성하였다.

연산자의 종류를 `if`와 `else if` 조건문으로 구분하였다. 거듭제곱은 반복문으로 같은 수를 여러 번 곱하는 방식으로 계산하였다.

### 어려웠던 점 및 해결 방법

덧셈, 뺄셈, 곱셈, 나눗셈을 구현하는 과정에서는 큰 어려움이 없었다. 하지만 거듭제곱을 구현할 때 두 번째 입력값인 `num2`가 실수형이어서 반복문의 횟수로 사용하는 과정에서 어려움이 있었다.

이를 해결하기 위해 반복문의 조건에서 `(int)num2`와 같이 형변환하여 정수로 사용하였다. 또한 곱셈 결과를 누적하는 `exp` 변수의 초기값을 지정하지 않아 예상과 다른 값이 출력되는 문제도 발생하였다. `exp`의 초기값을 1로 설정하여 문제를 해결하였다.

## 과제 5. 대칭 별 모양 출력

### 구현 방법

사용자로부터 숫자를 입력받아 좌우가 대칭인 별 모양을 출력하였다. 모양을 윗부분과 아랫부분으로 나누고, 중첩 `for` 반복문으로 별과 공백의 개수를 조절하였다.

윗부분에서는 별이 증가하고 공백이 감소하며, 아랫부분에서는 별이 감소하고 공백이 증가하도록 작성하였다.

### 어려웠던 점 및 해결 방법

일곱 개의 과제 중 가장 오랜 시간이 걸린 문제였다. 처음에는 주어진 별 모양 전체를 한 번에 출력해야 한다고 생각하여 별과 공백의 규칙을 찾는 것이 어려웠다.

하지만 전체 모양을 별의 개수가 증가하는 부분과 감소하는 부분으로 나누어 생각하였다. 즉, 윗부분은 삼각형 형태로, 아랫부분은 역삼각형 형태로 구분하여 각각 반복문을 작성하였다. 모양을 두 부분으로 나누자 각 줄에서 별과 공백이 변화하는 규칙을 찾을 수 있었고 문제를 해결할 수 있었다.

## 과제 6. 속이 빈 피라미드 출력

### 구현 방법

입력한 숫자를 높이로 가지는 속이 빈 피라미드를 출력하였다. 각 줄의 앞쪽에는 공백을 출력하고, 피라미드의 양쪽 테두리에 별을 출력하였다.

첫 번째 줄에는 별 하나를 출력하고, 중간 줄에는 두 개의 별과 내부 공백을 출력하였다. 마지막 줄에는 `2 × n - 1`개의 별을 출력하였다.

### 어려웠던 점 및 해결 방법

과제 5를 풀지 않고 과제 6을 먼저 해결했다면 과제 5보다 더 오랜 시간이 걸렸을 것이라고 생각한다. 하지만 과제 5에서 중첩 반복문으로 별과 공백의 개수를 조절하는 방법을 익힌 후 과제 6을 수행하여 어느 정도 힌트를 얻을 수 있었다.

과제 5와 마찬가지로 각 줄에 필요한 앞쪽 공백과 별의 개수를 먼저 계산하였다. 이후 첫 번째 줄, 중간 줄, 마지막 줄의 출력 방식이 다르다는 점을 확인하고 조건문으로 각각 구분하여 해결하였다.

## 과제 7. 순열과 조합 계산하기

### 구현 방법

사용자로부터 `n`과 `r`을 입력받아 순열, 중복순열, 조합, 중복조합을 계산하였다.

반복문으로 `n!`, `(n-r)!`, `r!`, `(n+r-1)!`, `(n-1)!`을 계산한 뒤 각각의 공식에 적용하였다. 잘못된 범위의 값이 입력되면 오류 메시지를 출력하도록 작성하였다.

### 어려웠던 점 및 해결 방법

과제 4에서 거듭제곱을 구현할 때 반복문으로 같은 수를 여러 번 곱하는 방법을 사용하였다. 팩토리얼도 일정한 범위의 수를 반복하여 곱한다는 점에서 거듭제곱 구현 방법과 비슷하다고 생각하였다.

각 공식에 필요한 팩토리얼 값을 확인한 뒤 반복문으로 차례대로 곱하여 계산하였다. 과제 4에서 사용한 누적 곱셈 방식을 활용할 수 있었기 때문에 큰 어려움 없이 해결할 수 있었다.

# 종합 느낀점

이번 과제를 수행하기 전까지 나는 C언어에 능숙한 편이 아니었기 때문에, 스스로 문제를 해결하는 능력을 기르는 것을 가장 중요한 목표로 삼았다. 따라서 과제 1부터 과제 7까지 수행하는 동안 LLM에 대한 의존을 최대한 줄이고, 배운 내용을 바탕으로 직접 코드를 작성하려고 노력하였다.

혼자서 해결하기 어려운 부분이 있을 때도 전체 코드를 요청하기보다는 이해되지 않는 몇 줄의 코드나 문제 해결 방향에 대한 힌트만 받아 과제를 완성하였다. 이러한 과정을 통해 정답을 바로 얻는 것보다 직접 고민하고 오류를 수정하는 과정이 실력 향상에 더 큰 도움이 된다는 것을 느꼈다.

앞으로 진행되는 C언어 교육에서도 LLM에 지나치게 의존하지 않고, 먼저 스스로 충분히 고민한 후 필요한 경우에만 보조 도구로 활용할 것이다. 이를 통해 C언어 문법에 대한 이해와 프로그램을 직접 설계하고 구현하는 능력을 꾸준히 향상하고 싶다.
